# Dasar Teori

<p align="justify">
Git merupakan sistem kontrol versi terdistribusi (Distributed Version Control System / DVCS) yang digunakan untuk mengelola perubahan pada file atau kode program selama proses pengembangan perangkat lunak. Git memungkinkan banyak pengembang bekerja secara bersamaan dalam satu proyek tanpa saling menimpa perubahan yang dibuat. Dengan Git, setiap perubahan yang dilakukan pada proyek dapat dicatat dalam bentuk commit, sehingga riwayat pengembangan dapat dilihat kembali, dibandingkan, maupun dikembalikan ke versi sebelumnya jika terjadi kesalahan. Git juga mendukung konsep branching yang memungkinkan pengembang membuat cabang pengembangan baru tanpa mengganggu kode utama.
</p>

<p align="justify">
Selain itu, Git sering digunakan bersama dengan platform repositori daring seperti GitHub, GitLab, dan Bitbucket untuk mempermudah kolaborasi dan penyimpanan proyek secara online. Melalui platform tersebut, pengembang dapat menyimpan kode sumber, melacak perubahan, melakukan kolaborasi tim, serta mengelola pengembangan proyek dengan lebih terorganisir. Dengan berbagai fitur yang dimilikinya, Git menjadi salah satu alat yang sangat penting dalam proses pengembangan perangkat lunak modern.
