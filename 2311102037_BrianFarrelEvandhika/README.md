# Dasar Teori

Git merupakan sistem kontrol versi (Version Control System/VCS) yang berfungsi untuk melacak, menyimpan, serta mengelola setiap perubahan file dalam sebuah proyek, khususnya dalam pengembangan perangkat lunak. Git diciptakan oleh Linus Torvalds pada tahun 2005 guna mendukung pengembangan sistem operasi Linux. Melalui Git, setiap perubahan kode disimpan dalam bentuk commit, sehingga riwayat perubahan dapat ditinjau, dibandingkan, maupun dikembalikan ke versi sebelumnya dengan mudah.

Di sisi lain, GitHub adalah platform berbasis web yang digunakan untuk menyimpan dan mengelola repository Git secara online. GitHub menyediakan berbagai fitur kolaborasi seperti manajemen branch, pull request, serta pelacakan perubahan kode, sehingga memudahkan tim dalam bekerja bersama dalam satu proyek. Selain itu, platform ini juga memungkinkan pengguna untuk berbagi kode dan mengelola proyek secara lebih terstruktur.

Dengan memanfaatkan Git sebagai sistem kontrol versi dan GitHub sebagai layanan hosting repository, proses pengembangan perangkat lunak menjadi lebih efisien, terorganisir, serta mendukung kolaborasi tim dalam membangun dan memelihara proyek.
