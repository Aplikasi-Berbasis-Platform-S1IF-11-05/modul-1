# Dasar Teori

<p align="justify">
Instalasi perangkat lunak merupakan proses pemasangan suatu aplikasi atau program ke dalam sistem komputer agar dapat digunakan oleh pengguna. Proses ini biasanya melibatkan penyalinan file program ke dalam sistem, pengaturan konfigurasi awal, serta penyesuaian dengan sistem operasi yang digunakan. Melalui proses instalasi yang benar, perangkat lunak dapat berjalan dengan optimal dan mendukung berbagai kebutuhan pengguna dalam menjalankan aktivitas komputasi maupun pengembangan perangkat lunak.
</p>

<p align="justify">
Git merupakan sistem <i>version control</i> terdistribusi yang digunakan untuk mengelola perubahan pada <i>source code</i> selama proses pengembangan perangkat lunak. Dengan menggunakan Git, pengembang dapat mencatat setiap perubahan yang dilakukan pada file sehingga riwayat pengembangan proyek dapat dilihat kembali kapan saja. Selain itu, Git juga memungkinkan beberapa pengembang untuk bekerja secara bersamaan pada proyek yang sama tanpa harus khawatir perubahan yang dilakukan saling menimpa satu sama lain.
</p>

<p align="justify">
Dalam penggunaannya, Git biasanya terhubung dengan layanan penyimpanan <i>repository</i> berbasis cloud seperti <i>GitHub</i>. Platform ini memungkinkan pengguna untuk menyimpan, mengelola, serta membagikan proyek kepada orang lain secara daring. Melalui layanan tersebut, pengembang dapat melakukan kolaborasi, mengatur versi kode program, serta melakukan pengelolaan proyek dengan lebih terstruktur menggunakan berbagai perintah pada <i>command line</i> seperti <b>git init</b>, <b>git add</b>, <b>git commit</b>, <b>git pull</b>, dan <b>git push</b>.
</p>
